# Сад «Цифровой геноцид» — плоская версия для GitHub Pages

Та же самая программа, что уже работает на Netlify, но **без подпапок** —
все файлы лежат рядом, на одном уровне. Специально сделано так, чтобы
загрузка через веб-интерфейс GitHub (drag-and-drop в браузере) ничего не
могла сломать — плющить уже нечего.

## Файлы

```
index.html
style.css
app.js
graph.json      (граф связей)
notes.json      (полный текст всех заметок, ~21 МБ)
parse_vault.py  (скрипт пересборки данных из Obsidian, см. отдельно)
```

## Как выложить на GitHub Pages

1. Создайте репозиторий (или очистите старый — удалите там всё).
2. **Add file → Upload files** — перетащите туда все 6 файлов из этого
   архива **по отдельности как файлы**, не как папку. Так как здесь нет
   вложенных папок вообще, результат будет одинаковым, что бы вы ни
   перетаскивали — сама папка или её содержимое.
3. Дождитесь, пока `notes.json` (21 МБ) полностью загрузится и появится
   в списке файлов коммита — не закрывайте вкладку раньше времени.
4. **Commit changes**.
5. Settings → Pages → Source: **Deploy from a branch** → Branch: **main**,
   папка **/(root)** → Save.
6. Подождите 1–2 минуты, откройте появившуюся ссылку.

## Если снова 404

Откройте прямо в адресной строке:
```
https://<ваш-логин>.github.io/<репозиторий>/notes.json
```
Если там страница «Page not found» — значит файл не долетел при загрузке
(GitHub веб-загрузчик иногда молча обрывает большие файлы). Решение —
загрузить `notes.json` ещё раз отдельно, одним файлом, и дождаться полной
закачки перед коммитом.

## Обновление данных после правок в Obsidian

```
python3 parse_vault.py --vault путь/к/wiki13 --out .
```
Скрипт положит новые `graph.json` и `notes.json` рядом с собой (в корень) —
загрузите их поверх старых на GitHub.
